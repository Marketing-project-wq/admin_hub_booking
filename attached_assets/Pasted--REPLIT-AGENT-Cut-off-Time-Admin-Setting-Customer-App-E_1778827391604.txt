# REPLIT AGENT — Cut-off Time: Admin Setting + Customer App Enforcement

> **Context:** Fitur baru cut-off time untuk arena class booking.
> **Logic:** Customer tidak bisa booking kelas jika waktu sekarang sudah melewati `(start_time - cutoff_minutes)`.
> **Default global:** 60 menit (dari `arena_booking_units.cutoff_minutes`).
> **Override:** Bisa di-set per jadwal via `arena_class_schedules.cutoff_minutes` (NULL = pakai default unit).
>
> **Prereq:** SQL migration sudah dijalankan (kolom `cutoff_minutes` sudah ada di kedua tabel).

## BAGIAN 2 — CUSTOMER BOOKING APP

### 2a. Fetch jadwal — filter jadwal yang sudah melewati cut-off

**File:** halaman/komponen yang fetch `arena_class_schedules` untuk ditampilkan ke customer.

Tambah filter: hanya tampilkan jadwal yang booking deadline-nya belum lewat.

```javascript
// Fetch schedules dengan filter cut-off
const fetchSchedules = async (selectedDate) => {
  const nowISO = new Date().toISOString()

  const { data: schedules, error } = await supabase
    .from('arena_class_schedules')
    .select(`
      id, schedule_date, start_time, end_time, instructor, quota,
      cutoff_minutes,
      class_type:arena_class_types(id, name, color, price_guest, duration_min),
      unit:arena_booking_units(id, name, cutoff_minutes)
    `)
    .eq('schedule_date', selectedDate)
    .eq('is_cancelled', false)
    .order('start_time', { ascending: true })

  if (error || !schedules) return []

  // Filter client-side: hanya jadwal yang booking deadline belum lewat
  const now = new Date()
  const filtered = schedules.filter(s => {
    // Effective cutoff: override schedule atau default unit
    const cutoffMins = s.cutoff_minutes ?? s.unit?.cutoff_minutes ?? 60

    // Booking deadline = schedule_date + start_time - cutoff_minutes
    const [hours, minutes] = s.start_time.split(':').map(Number)
    const classStart = new Date(`${s.schedule_date}T${s.start_time}`)
    const deadline = new Date(classStart.getTime() - cutoffMins * 60 * 1000)

    return now < deadline
  })

  return filtered
}
```

### 2b. Tampilkan info cut-off di UI customer (opsional tapi recommended)

Di card jadwal, tampilkan deadline booking:

```jsx
const getCutoffDeadline = (schedule) => {
  const cutoffMins = schedule.cutoff_minutes ?? schedule.unit?.cutoff_minutes ?? 60
  const classStart = new Date(`${schedule.schedule_date}T${schedule.start_time}`)
  const deadline = new Date(classStart.getTime() - cutoffMins * 60 * 1000)
  return deadline
}

// Di dalam card jadwal:
const deadline = getCutoffDeadline(schedule)
const isClosingSoon = (deadline - new Date()) < 30 * 60 * 1000 // < 30 menit lagi

{isClosingSoon && (
  <p style={{ color: '#E53E3E', fontSize: 12, margin: '4px 0 0' }}>
    ⚠️ Booking tutup {deadline.toLocaleTimeString('id-ID', { hour: '2-digit', minute: '2-digit' })} WIB
  </p>
)}
```

### 2c. Validasi saat submit booking
